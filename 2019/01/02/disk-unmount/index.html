macOS 下硬盘和U盘卸载遇到的问题

2019-01-02
macos

刚上大学时用 Windows 养成了 U盘即插即拔的坏习惯，拔的时候为了省事儿也不先卸载，因为每次拔的时候不卸载并没有对下次插入的读写造成任何影响，索性就觉得拔之前的卸载操作是多余的，意识不到卸载操作的重要性，这也为以后出现问题埋下祸根。我 Mac 是用 mounty 来挂载写保护的 NTFS 硬盘为读写模式，最开始的这种不当操作导致我无法重新挂载硬盘，但在我每次拔之前用 mounty 卸载后仍出现了最近发生的匪夷所思问题：

首先是硬盘数据的“丢失”，原先下载好的视频文件“丢失”了一部分，即在 Finder 中只能看到部分文件，起初怀疑文件被隐藏了，用 ls -a 仍然显示不出来，但当我用 aria2 重新下载种子时，却是在 seeding 而不是 downloading，所以我判断文件依然存在，于是就出现了下面的截图

能 cd 进入 ls -a 无法显示的目录！这是什么操作！！！

用 Windows 电脑挂载硬盘，打开后显示文件夹损坏，无法读取，然后用 chkdsk 命令修复硬盘后重新挂载到 Mac 上成功将文件显示出来：

回忆了一下，觉得是在弹出硬盘之前忘了用 Finder 推出硬盘才导致的上述问题出现，即下次不仅需要用 mounty 卸载，还需要在 Finder 中卸载
